ProteinBlender를 설치하면 Blender 실행 시 전용 워크스페이스 탭("Protein Blender")이 자동으로 생성됩니다. 이 워크스페이스는 왼쪽 70%에 3D Viewport, 오른쪽 30%에 Properties Editor가 배치되어 있습니다.
ProteinBlender의 모든 패널은 Properties Editor의 Scene 탭(원뿔 🔻 아이콘)에 있습니다. 패널을 못 찾겠으면 페이지 하단의 패널 위치 가이드를 참고하세요.

1 단백질 구조 임포트

Protein Import 패널에서 단백질 구조를 불러옵니다. 세 가지 방법이 있습니다.

방법입력설명
로컬 파일 .pdb / .cif 폴더 아이콘 클릭 → 파일 선택. OpenFold3 결과 등 로컬 구조
RCSB PDB 4자리 PDB ID 예: 1CRN (크램빈), 4HHB (헤모글로빈), 3SN6 (GPCR+Gs)
AlphaFold DB UniProt ID AlphaFold 예측 구조를 직접 다운로드

따라하기

  1. Properties Editor → Scene 탭(원뿔 아이콘)에서 Protein Import 패널 찾기
  2. PDB ID 입력란에 1CRN 입력
  3. "Download" 클릭
  4. 3D 뷰포트에 Cartoon 스타일로 단백질이 나타남
처음 연습할 때는 1CRN (46 잔기, 아주 작음)이 가장 좋습니다. 큰 단백질 (1000+ 잔기)은 임포트에 30~60초 걸릴 수 있습니다.
"Failed to import" 에러가 나타나면 문제 해결 가이드를 참고하세요. 가장 흔한 원인은 Blender의 "Allow Online Access" 설정이 꺼져 있는 경우입니다.

2 시각화 스타일 조정

Visual Setup 패널에서 색상과 분자 표현 방식을 바꿉니다.

스타일용도적합한 상황
Cartoon 알파 헬릭스, 베타 시트 강조 논문/발표용 전체 구조 (기본값)
Surface 분자 표면, 결합 포켓 시각화 약물-단백질 상호작용 설명
Ribbon 간략한 백본 표시 큰 복합체의 간결한 표현
Sticks 결합을 막대로 표시 결합 패턴, 리간드 구조
Ball & Stick 원자 + 결합 상세 표현 활성 부위 클로즈업
Spheres van der Waals 공간 충전 모델 분자 크기/부피 비교

따라하기

  1. 셰이딩 모드 전환: 3D Viewport 우측 상단 → Material Preview (세 번째 구체 아이콘) 또는 Z 키
    Solid 모드에서는 색상/스타일 변경이 보이지 않습니다. 반드시 Material Preview 이상으로 전환하세요.
  2. Protein Outliner에서 색상을 바꿀 항목의 체크박스 선택
  3. Visual Setup 패널로 이동
  4. 컬러 피커에서 원하는 색상 클릭 — 별도 버튼 없이 즉시 적용됩니다 (라이브 업데이트)
  5. Representation 드롭다운에서 원하는 스타일 변경

색상 적용의 계층 구조

어떤 항목을 선택하느냐에 따라 색상이 적용되는 범위가 달라집니다:

선택 대상색상 적용 범위
Protein (최상위)단백질 전체 + 모든 체인 + 모든 도메인
Chain해당 체인 + 소속 도메인 전체
Domain해당 도메인만
색상이 안 바뀌는 경우? → 뷰포트 셰이딩 모드를 확인하세요!
ProteinBlender의 색상은 Geometry Nodes로 적용되므로 Solid 모드에서는 색상 변경이 보이지 않습니다.
3D Viewport 오른쪽 상단에서 세 번째 구체 아이콘 (Material Preview)을 클릭하거나, Z 키로 셰이딩 모드를 전환하세요.
리간드(HEM, 약물 등)는 ProteinBlender에서 독립적으로 스타일 변경이 불가합니다.
ProteinBlender의 도메인 시스템은 단백질 잔기 범위만 지원합니다. 리간드/헤테로아톰은 Protein Outliner에 표시되지 않으며, 별도로 스타일링하려면 Blender의 Geometry Nodes 에디터에서 MolecularNodes의 Select Ligand_ 또는 Select Res Name 노드로 리간드를 선택한 뒤 Style Ball and Stick 등의 스타일 노드를 연결해야 합니다.
도메인별로 다른 스타일을 적용할 수 있습니다. Domain Maker로 체인을 여러 도메인으로 분할한 뒤, 각 도메인에 다른 색상/스타일을 지정하세요.

3 Puppet 만들기 (그룹핑)

Puppet은 여러 체인/도메인을 하나로 묶어서 함께 움직이게 하는 그룹입니다. 애니메이션의 핵심 단위입니다.

따라하기

  1. Protein Outliner에서 묶고 싶은 체인/도메인의 체크박스 선택
  2. Protein Puppet Maker 패널 열기
  3. "Create New Puppet" 클릭
  4. 알아보기 쉬운 이름 입력 (예: enzyme_A)
조작단축키설명
이동 (Grab)GPuppet 선택 후 마우스로 위치 이동
회전 (Rotate)RPuppet 선택 후 회전
크기 (Scale)SPuppet 선택 후 크기 조절
축 제한G→X/Y/Z특정 축으로만 이동/회전 제한
각 체인/도메인은 하나의 Puppet에만 속할 수 있습니다. Puppet에서 멤버를 빼려면 삭제 후 재생성해야 합니다.

4 Pose 저장 (컨포메이션 스냅샷)

Pose는 Puppet들의 위치/회전/크기를 저장하는 스냅샷입니다. 서로 다른 생물학적 상태를 기록해 두었다가 전환할 수 있습니다.

따라하기

  1. Puppet을 원하는 위치로 배치 (G, R, S)
  2. Pose Library 패널 열기
  3. "Create Pose" 클릭
  4. 이름 입력 (예: Open_State) → 포함할 Puppet 선택 → OK
  5. Puppet을 다른 위치로 이동 후 다시 "Create Pose" → Closed_State
버튼기능
Apply저장된 Pose로 즉시 복원
Capture현재 위치로 기존 Pose 업데이트 (덮어쓰기)
DeletePose 삭제 (Puppet은 유지됨)
Pose는 위치/회전/크기만 저장합니다. 색상이나 시각화 스타일은 포함되지 않습니다. 색상 전환 애니메이션이 필요하면 키프레임 단계에서 Color 옵션을 사용하세요.
생물학적 상태에 맞는 이름을 쓰세요: ATP_Bound, Substrate_Free, Transition_State 등. Pose는 Blender 파일(.blend) 안에 Scene별로 저장되며, 다른 .blend 파일로 전달되지 않습니다.

5 키프레임 애니메이션

Pose 사이를 부드럽게 전환하는 애니메이션을 만듭니다. Blender가 키프레임 사이를 자동으로 보간합니다.

키프레임 삽입 방법 (2가지)

방법위치설명
방법 A (권장) Properties Editor → Scene 탭 → Animate Scene 패널 "Create Keyframe" 버튼 클릭. 대화 상자에서 기록할 항목을 선택합니다:
  • Location — 위치
  • Rotation — 회전
  • Scale — 크기
  • Pose — 도메인 멤버들의 배치
  • Color — 색상 전환 (체크 시 색상 애니메이션 가능)
방법 B (Blender 기본) 3D Viewport 위에서 K 키 Insert Keyframe Menu가 열립니다 → Location, Rotation, Scale 중 선택.
※ Blender 4.1+에서는 I 키가 메뉴 없이 LocRotScale을 한꺼번에 삽입합니다. 개별 채널만 키프레임하려면 K 키를 사용하세요.

기본 애니메이션 만들기

  1. 타임라인에서 프레임 1로 이동
  2. Puppet 선택 → 시작 위치(Pose) 배치
  3. Animate Scene 패널 → "Create Keyframe" 클릭 (시작점 기록)
  4. 타임라인에서 프레임 60으로 이동
  5. Puppet을 새 위치로 이동/회전
  6. 다시 "Create Keyframe" 클릭 (끝점 기록)
  7. Space로 재생하여 결과 확인
키프레임이란? "이 프레임에서 오브젝트는 이 위치/회전/크기에 있어야 한다"를 기록하는 것입니다. 두 키프레임 사이를 Blender가 자동으로 보간하여 부드러운 움직임을 만들어줍니다.

Pose 전환 애니메이션

  1. 프레임 1 → Pose Library에서 Open_State Apply → "Create Keyframe"
  2. 프레임 60 → Closed_State Apply → "Create Keyframe"
  3. 재생하면 열림→닫힘 구조 변화가 부드럽게 전환됨
키프레임 대상설명
위치 / 회전 / 크기Puppet의 3D 공간 이동
Pose 전환서로 다른 컨포메이션 간 전환
색상 변화시간에 따른 색상 전환 효과

애니메이션 재생 방법

방법설명
Space 키 마우스 커서가 반드시 3D Viewport(단백질이 보이는 영역) 위에 있어야 합니다.
사이드바나 다른 패널 위에서는 작동하지 않습니다.
타임라인 ▶ 버튼 화면 하단 타임라인 바의 재생 버튼 클릭
재생이 안 되는 경우 체크리스트:
① Shift+←로 프레임 1로 돌아가세요 (마지막 프레임에서는 움직임이 보이지 않음)
② 마우스 커서를 3D Viewport 위로 옮긴 후 Space
③ 다른 창(Preferences 등)이 열려 있으면 닫으세요
④ Preferences → Keymap → Spacebar Action이 "Play"로 설정되어 있는지 확인

반복(Cyclic) 애니메이션 설정

애니메이션이 끝나면 자동으로 프레임 1로 돌아가서 반복 재생되게 하려면 두 곳을 모두 설정해야 합니다.

① Blender 프레임 범위 설정 (필수)

Blender 자체의 재생/렌더링 범위를 키프레임 범위와 일치시켜야 합니다.

  1. 화면 오른쪽 Properties 패널에서 프린터 아이콘 (Output Properties) 클릭
  2. 맨 위 Frame Range 섹션에서 End Frame을 120으로 변경
  3. (기본값이 250이므로, 120으로 바꾸지 않으면 프레임 120 이후에도 재생이 계속됩니다)

② 반복 재생 활성화 (선택)

방법설명
ProteinBlender Animate Scene 패널 → Manual Frame Range (Start/End를 1~120) 설정 + "Cycle Animation" 체크
Graph Editor (확실한 방법) Shift+F6으로 Graph Editor 열기 →
A로 모든 커브 선택 → 메뉴 Channel → Extrapolation Mode → Make Cyclic (F-Modifier)
커브 양 끝에 점선이 표시되면 성공
ProteinBlender의 "Manual Frame Range"와 Blender의 "Frame Range"는 별개입니다!
ProteinBlender의 설정만으로는 Blender의 타임라인 End가 바뀌지 않을 수 있습니다. 반드시 ①번(Blender 프레임 범위)도 직접 설정하세요.
Output Properties의 End Frame 값이 키프레임 마지막 프레임(예: 120)과 동일한지 확인하세요.
Graph Editor (Shift+F6): 보간 곡선 정밀 제어
Dope Sheet (Shift+F2): 전체 키프레임 타임라인 관리

6 렌더링 & 출력

애니메이션이 완성되면 Blender의 렌더링 기능으로 영상을 출력합니다.

렌더링 설정

Properties Editor → Output Properties 탭 (프린터 🖨️ 아이콘)에서 아래 항목을 설정합니다.

섹션항목설정값
Format Resolution X / Y 발표용: 1920 × 1080, SNS/웹: 1280 × 720
Frame Rate 24 fps (기본값, 영화 느낌) 또는 30 fps
Frame Range Frame Start / End Start: 1, End: 마지막 키프레임 (예: 120)
Output 출력 경로 저장할 폴더 경로 지정 (폴더 아이콘 클릭)
Output
(File Format)		 File Format FFmpeg Video 선택
Color RGB (FFmpeg Video 선택 시 BW / RGB만 표시됨)
Encoding Container MPEG-4로 변경 (기본값 Matroska → MPEG-4로 바꾸세요)
Video Codec H.264 (기본값, 가장 호환성 좋음)
Output Quality Medium Quality (기본값) 또는 High Quality (논문용)
  1. 위 설정 완료 후 → 메뉴 Render → Render Animation (Ctrl+F12)
Container는 반드시 Matroska(기본값) → MPEG-4로 변경하세요!
Matroska(.mkv)는 PowerPoint/Keynote에서 재생 불가, 일부 브라우저 미지원, 학술지 투고에도 부적합합니다.
MPEG-4(.mp4)는 모든 OS, 브라우저, 발표 소프트웨어, YouTube, 학술지에서 호환됩니다.
프리뷰는 Render Engine을 EEVEE로, 최종본은 Cycles로 하면 품질과 속도를 균형 있게 조절할 수 있습니다. Mac에서는 Metal GPU 가속이 Cycles에서 잘 작동합니다.

예제 A-1: 크램빈(1CRN) 360도 회전

난이도: 입문 — 소요시간: ~10분 — 크램빈은 46 잔기의 작은 단백질로 연습에 최적

  1. 임포트: Protein Import 패널 → PDB ID에 1CRN 입력 → Download
  2. 색상 변경: Visual Setup에서 원하는 색상 선택 (컬러 피커 클릭 시 즉시 적용)
    ※ 뷰포트 셰이딩을 Material Preview (Z)로 전환해야 색상이 보입니다
  3. Puppet 생성: Outliner에서 체인 선택 → Create New Puppet → 이름: crambin
  4. 프레임 1: Puppet 선택 → Animate Scene → "Create Keyframe" (시작점 기록)
  5. 프레임 60으로 이동 → R → Z → 180 입력 → Enter → "Create Keyframe"
  6. 프레임 120으로 이동 → R → Z → 180 입력 → Enter → "Create Keyframe"
  7. Shift+←로 프레임 1로 돌아간 후 → Space로 재생 → 360도 회전 확인!
  8. 매끄러운 회전: Graph Editor 열기 (Shift+F6) → 회전 커브 선택 → T → Linear로 변경
왜 360도를 한 번에 입력하면 안 되나요?
Blender는 내부적으로 쿼터니언(Quaternion)을 사용하여 회전을 저장합니다. 360도 회전 = 0도 회전과 수학적으로 동일하므로, 프레임 1과 프레임 120의 회전값이 같아져서 아무 움직임도 발생하지 않습니다. 반드시 180도씩 2단계로 나누어 입력하세요.
처음에는 Ease-In/Out 보간이 기본이라 시작과 끝이 느려집니다. Linear로 바꾸면 일정한 속도로 회전합니다. 루프 영상에 적합합니다.

예제 A-2: 터닝테이블 렌더링 (Blender 기본 키프레임)

난이도: 입문 — ProteinBlender Puppet 없이 Blender 기본 키프레임으로 단백질을 회전. 조명·카메라 설정까지 포함한 완전한 렌더링 예제

  1. 임포트: 1CRN Download (위 예제와 동일)
  2. 뷰포트 셰이딩 모드 변경:
    • 3D Viewport 우측 상단에서 세 번째 구체 아이콘 (Material Preview) 클릭, 또는 Z 키로 전환
    • 기본 Solid 모드에서는 단백질 색상과 재질이 보이지 않습니다. Material Preview 이상에서만 실제 색상이 표시됩니다.
  3. 단백질 중심 맞추기:
    • Outliner(좌측 상단 오브젝트 목록)에서 단백질 오브젝트(예: Chain_A_0_1_44) 선택
    • 3D Viewport에서 우클릭 → Set Origin → Origin to Geometry (기준점을 단백질 중심으로)
    • Alt+G → 위치를 원점(0,0,0)으로 리셋
    • 이렇게 하면 단백질이 정확히 원점에서 제자리 회전합니다
  4. 조명 추가:
    • Shift+A → Light → Sun
    • Sun 조명은 위치와 무관하게 전체를 균일하게 비춥니다
    • 밝기 조절: Sun 선택 → Properties Editor → Light 탭(전구 아이콘) → Strength: 3~5
  5. 카메라 위치 조정:
    • Camera 선택 → Numpad 0으로 카메라 뷰 진입
    • N 키 → 사이드바 → View 탭 → Lock Camera to View 체크
    • 마우스 휠로 줌 조절, 가운데 버튼 드래그로 앵글 조절 — 단백질이 적당한 크기로 보이게
    • 완료 후 Lock Camera to View 체크 해제 (해제하지 않으면 뷰 조작 시 카메라가 계속 이동함)
    • 카메라 뷰에서 나가려면: Numpad 5 또는 마우스 가운데 버튼 드래그
  6. 회전 키프레임 설정 (Blender 4.1+ 기준):
    • 단백질 오브젝트(Chain_A_0_1_44) 선택
    • 프레임 1 → K 키 → Rotation 선택 (시작점 기록)
      ※ Blender 4.1+에서는 I가 아닌 K 키로 Insert Keyframe Menu를 엽니다. I 키는 메뉴 없이 Location+Rotation+Scale을 한꺼번에 삽입합니다.
    • 프레임 120 → R → Z → 360 → Enter → K → Rotation
  7. 등속 회전 설정:
    • 하단 타임라인 에디터 좌측의 에디터 타입 아이콘 클릭 → Graph Editor 선택
      (또는 기존 에디터 영역에서 에디터 타입 드롭다운 변경)
    • A로 모든 커브 선택 → T → Linear — 일정 속도 회전
  8. 렌더링 설정 & 출력:
    • Properties Editor → Output Properties(프린터 아이콘) → End Frame: 120
    • File Format: FFmpeg Video, Container: MPEG-4
    • Ctrl+F12로 렌더링
렌더링 결과가 어둡거나 단백질이 안 보이면:
① 조명이 있는지 확인 (Outliner에 Sun/Light가 있어야 함)
② 카메라가 단백질을 향하고 있는지 Numpad 0으로 확인
③ Material이 반투명하면: 단백질 선택 → Material Properties(구 아이콘) → Subsurface → Weight를 0으로
더 높은 렌더링 품질: Render Properties(카메라 아이콘) → Render Engine을 Cycles로 변경, Device를 GPU Compute로 설정. Mac에서는 Metal 가속이 자동 적용됩니다.

예제 B-1: c-Src 키나제 — 도메인 분할과 색상 구분

단일 체인 단백질을 생물학적 도메인(SH3 / SH2 / Kinase)으로 분할하여 영역별로 다른 색상을 적용합니다.

왜 c-Src인가? c-Src은 최초로 발견된 proto-oncogene 산물로, 하나의 체인에 3개의 뚜렷한 도메인(SH3, SH2, Kinase)이 있습니다. 비활성 상태에서 SH3와 SH2가 키나제 도메인 위로 접혀 자기억제(autoinhibition) 구조를 형성합니다 — 도메인 색 구분으로 이 구조적 특성을 직관적으로 보여줄 수 있습니다.
  1. 임포트: PDB ID 2SRC → Download (450 잔기, 단일 체인)
  2. 셰이딩 모드 전환: 3D Viewport 우측 상단 → Material Preview (세 번째 구체 아이콘) 또는 Z 키
    Solid 모드에서는 색상 변경이 보이지 않습니다
  3. 전체 색상 변경: Protein Outliner에서 Chain A 체크박스 선택 → Visual Setup → 컬러 피커에서 연한 회색 선택 (기본 바탕색)
  4. 도메인 분할: Domain Maker로 3개 생물학적 도메인을 나눕니다
    • Protein Outliner에서 Chain A 선택
    • Domain Maker 패널 열기 (선택한 체인이 있어야 표시됨)
    • 첫 번째 분할 — SH3 도메인: "Split Chain" 버튼 → 잔기 범위 84~145 지정
      → 3개 도메인 자동 생성: N-terminal(~83) / SH3(84~145) / 나머지(146~)
    • 두 번째 분할 — 나머지 도메인에서 SH2 분리: 나머지 도메인 선택 → "Split Chain" → 잔기 범위 151~248 지정
      → 링커(146~150) / SH2(151~248) / Kinase(249~) 으로 분할
  5. 도메인별 색상 적용:
    도메인잔기 범위추천 색상기능
    SH384 ~ 145초록색 (#50C878)프롤린-리치 모티프 인식
    SH2151 ~ 248주황색 (#E88D3F)인산화 티로신 결합
    Kinase270 ~ 523파란색 (#4A90D9)촉매 도메인 (N-lobe + C-lobe)
    링커/N-tail나머지연한 회색 (#B0B0B0)연결 부위
    Protein Outliner에서 각 도메인 체크박스 선택 → Visual Setup 컬러 피커에서 색상 변경 (즉시 적용)
  6. 결과: 하나의 체인이지만 SH3(초록), SH2(주황), Kinase(파란)로 명확히 구분된 구조가 보임
스타일도 다르게! SH3와 SH2를 Ribbon, Kinase를 Cartoon으로 설정하면 도메인 크기 차이가 더욱 극적으로 보입니다. Visual Setup → Representation 드롭다운에서 도메인별로 스타일을 변경할 수 있습니다.

예제 B-2: 헤모글로빈 — 멀티 체인 조립 애니메이션

4개의 서브유닛(α2β2)을 각각 다른 색상으로 칠하고, 분리 → 조립 애니메이션을 만듭니다.

왜 헤모글로빈인가? 교과서적인 α2β2 4량체로, 574 잔기밖에 되지 않아 가볍습니다. 실제 생물학에서도 α-β 이량체가 먼저 형성된 후 두 이량체가 결합하여 4량체를 만듭니다 — 이 조립 과정을 애니메이션으로 재현합니다.
  1. 임포트: PDB ID 4HHB → Download (574 잔기, 4체인)
  2. 셰이딩 모드 전환: Material Preview (Z 키)
  3. 체인별 색상 지정:
    체인서브유닛추천 색상
    Aα1파란색 (#4A90D9)
    Bβ1빨간색 (#D94A4A)
    Cα2하늘색 (#7AB8E0)
    Dβ2분홍색 (#E07A7A)
    Protein Outliner에서 체인 하나씩 선택 → Visual Setup 컬러 피커에서 색상 선택 (즉시 적용)
  4. Puppet 4개 생성: 각 체인을 개별 Puppet으로 생성
    • Chain A 선택 → Create Puppet → alpha1
    • Chain B 선택 → Create Puppet → beta1
    • (C → alpha2, D → beta2도 동일)
  5. "조립된 상태" Pose 저장:
    • 4개 Puppet 모두 현재 위치 그대로 유지
    • Pose Library → Create Pose → 이름: Assembled
  6. "분리된 상태" 만들기:
    • 각 Puppet을 G로 바깥쪽으로 이동 (서로 떨어지게)
    • alpha1: G → X → -3 → Enter
    • beta1: G → X → 3 → Enter
    • alpha2: G → Y → -3 → Enter
    • beta2: G → Y → 3 → Enter
    • Pose Library → Create Pose → 이름: Exploded
  7. 조립 애니메이션:
    • 프레임 1: Exploded Pose Apply → 모든 Puppet 선택 → "Create Keyframe"
    • 프레임 90: Assembled Pose Apply → "Create Keyframe"
    • 프레임 90~150: 정지 상태 유지 (같은 키프레임 복사)
  8. Space로 재생 → 4개 서브유닛이 날아와서 조립되는 애니메이션!
NOTE Pose는 위치/회전/크기만 저장합니다. 색상은 Visual Setup에서 별도로 설정해야 합니다.

예제 B-3: β2-아드레날린 수용체 + Gs 단백질 — GPCR 신호전달 복합체

GPCR 신호전달 복합체의 5개 체인을 기능별로 색상 구분하고, 수용체-G단백질 접합면으로 카메라 줌인 애니메이션을 만듭니다.

왜 3SN6인가? 2012년 노벨 화학상(Kobilka & Lefkowitz)의 핵심 구조입니다. GPCR인 β2-AR이 리간드(작용제)에 의해 활성화되면 세포 내 Gs 단백질과 결합하여 신호를 전달합니다. 5개 체인이 각각 다른 역할을 하므로 색상 구분만으로도 신호전달 메커니즘을 직관적으로 보여줄 수 있습니다.
  1. 임포트: PDB ID 3SN6 → Download (1,318 잔기, 5체인)
  2. 셰이딩 모드 전환: Material Preview (Z 키)
  3. 체인별 색상 지정:
    체인정체추천 색상역할
    AGαs주황색 (#E88D3F)G단백질 α 서브유닛 (GTPase)
    BGβ1초록색 (#50C878)G단백질 β 서브유닛 (WD40 프로펠러)
    CGγ2연두색 (#8FD68F)G단백질 γ 서브유닛
    Dβ2-AR파란색 (#4A90D9)수용체 (7TM 나선 번들)
    ENb35연한 회색 (#B0B0B0)나노바디 (결정화 보조 — 숨겨도 됨)
    Protein Outliner에서 체인 하나씩 선택 → Visual Setup 컬러 피커에서 색상 적용
  4. 나노바디 숨기기 (선택사항): Chain E(Nb35)는 결정화를 위한 보조 단백질입니다.
    Blender Outliner에서 해당 오브젝트의 눈 아이콘(👁) 클릭하여 숨기면 생물학적으로 의미 있는 구성만 보입니다.
  5. 카메라 설정 — 전체 뷰:
    • 카메라를 복합체 전체가 보이는 위치에 배치 (Numpad 0 → Lock Camera to View)
    • 프레임 1: 카메라 선택 → K → Location 키프레임 삽입
      ※ 카메라는 Puppet이 아니므로 Blender 기본 I 키로도 삽입 가능
  6. 줌인 — 수용체-G단백질 접합면:
    • 프레임 60: 카메라를 수용체(파란)와 Gα(주황)가 접촉하는 세포질 쪽 접합면으로 이동
    • 카메라 뷰에서 접합면이 화면 중앙에 크게 보이도록 조정
    • K → Location 키프레임 삽입
  7. 프레임 60~120: 줌인 상태에서 카메라를 약간 회전하여 접합면을 여러 각도로 보여줌
  8. Graph Editor에서 줌인 구간의 보간 곡선을 Ease-In으로 설정 → 자연스러운 감속
카메라 Depth of Field(피사계 심도) 추가: Camera Properties → Depth of Field → Focus Object를 수용체(Chain D)로 설정 → F-Stop 0.5~2.0. 배경의 G단백질이 살짝 흐려지면서 수용체에 시선이 집중됩니다.
Chain D(β2-AR)에 대해: 이 구조에는 결정화를 위해 수용체의 ICL3(세포질 루프3)에 T4 lysozyme이 삽입되어 있습니다. 수용체 중간에 비-GPCR 구조가 보일 수 있으나, 이는 실험적 산물입니다.

예제 B-4: TRPV1 채널 — 막단백질 4량체 Exploded View

캡사이신(고추 매운맛) 수용체 TRPV1의 4량체 구조를 서브유닛별로 분리하는 교육용 애니메이션.

왜 TRPV1인가? 2013년 cryo-EM "resolution revolution"의 상징적 구조이며 (2021 노벨 생리의학상, Julius & Patapoutian), C4 대칭의 호모 4량체로 4개 서브유닛이 동일한 구조를 가집니다. 리간드가 없어 ProteinBlender만으로 깔끔하게 시각화할 수 있습니다.
  1. 임포트: PDB ID 3J5P → Download (~2,400 잔기, 4체인 — 약간 시간 소요)
  2. 셰이딩 모드 전환: Material Preview (Z 키)
  3. 4개 서브유닛 색상 구분:
    체인추천 색상
    A파란색 (#4A90D9)
    B초록색 (#50C878)
    C주황색 (#E88D3F)
    D보라색 (#9B59B6)
    Protein Outliner에서 체인 선택 → Visual Setup 색상 적용
  4. Puppet 4개 생성: 각 서브유닛을 개별 Puppet으로 생성
    • Chain A → subunit_A, Chain B → subunit_B, (C, D 동일)
  5. Pose 2개 저장:
    • Tetramer_Intact: 원래 위치 그대로 저장
    • 각 Puppet을 채널 중심에서 방사형으로 바깥으로 이동 (G → 축 지정):
      subunit_A: G → X → 5 / subunit_B: G → X → -5 / subunit_C: G → Y → 5 / subunit_D: G → Y → -5
    • Tetramer_Exploded: 분리 위치 저장
  6. 타임라인 구성:
    프레임상태설명
    1~30Tetramer_Intact완전체 채널 구조를 보여줌
    30~90전환4개 서브유닛이 서서히 분리됨
    90~150Tetramer_Exploded분리 상태에서 회전하며 각 서브유닛 관찰
    150~210전환다시 합쳐짐
    210~240Tetramer_Intact마무리
  7. 각 전환 구간의 시작/끝에 "Create Keyframe" 삽입
대형 구조 성능 팁: TRPV1은 ~2,400 잔기 / ~18,000 원자로 뷰포트가 느려질 수 있습니다.
  • 작업 중에는 뷰포트 셰이딩을 Solid로 유지 (Material Preview보다 훨씬 빠름)
  • Representation을 Ribbon으로 설정하면 Cartoon보다 가벼움
  • 최종 렌더링만 Material Preview 또는 Cycles로 전환
  • EEVEE 렌더 엔진이 Cycles보다 빠름 (품질 vs 속도 트레이드오프)

부록: 문제 해결 (Troubleshooting)

"Failed to import" 에러

Download 버튼을 눌렀을 때 Blender 하단 상태 바에 Failed to import: 1CRN 같은 에러가 표시되는 경우:

1. Allow Online Access 확인 (가장 흔한 원인)

Blender 4.2+에서는 확장 프로그램의 인터넷 접근이 기본적으로 제한됩니다.

  1. Edit → Preferences (macOS: Blender → Settings)
  2. 왼쪽 메뉴에서 System 클릭
  3. Network 섹션에서 "Allow Online Access" 체크박스 활성화
  4. Preferences 창 닫기 → Download 재시도

2. 실제 에러 메시지 확인 (macOS)

ProteinBlender는 에러 상세 메시지를 콘솔에 출력하지만, macOS에서는 기본적으로 보이지 않습니다.

  1. Terminal 앱 열기 (Spotlight에서 "Terminal" 검색)
  2. 아래 명령어로 Blender 실행:
    /Applications/Blender.app/Contents/MacOS/Blender
  3. ProteinBlender에서 Download 다시 시도
  4. Terminal에 출력된 에러 메시지 확인 — Failed to import PDB 1CRN: ... 형태

3. 포맷 변경 시도

기본 포맷(PDB)으로 다운로드가 안 되면, Protein Import 패널에서 포맷을 mmCIF로 변경 후 재시도해 보세요. RCSB에서 일부 구조는 PDB 포맷이 제공되지 않을 수 있습니다.

4. MolecularNodes 업데이트 (databpy 충돌)

Blender 확장 목록에 MolecularNodes가 별도로 설치되어 있으면 databpy 버전 충돌이 발생할 수 있습니다. Terminal에 ImportError: cannot import name 'Domains' from 'databpy' 에러가 보이면:

  1. Edit → Preferences → Get Extensions
  2. MolecularNodes를 최신 버전으로 업데이트
  3. Blender 재시작 후 Download 재시도

ProteinBlender는 MolecularNodes를 내장하고 있어 별도 설치가 필수는 아니지만, 이미 설치되어 있다면 최신 버전으로 유지해야 충돌이 없습니다.

5. ProteinBlender 재설치

위 방법으로도 해결되지 않으면 ProteinBlender 확장을 제거 후 재설치해 보세요: Edit → Preferences → Get Extensions → Installed → ProteinBlender 제거 → 재설치
