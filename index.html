ProteinBlender를 설치하면 Blender 실행 시 전용 워크스페이스 탭("Protein Blender")이 자동으로 생성됩니다. 이 워크스페이스는 왼쪽 70%에 3D Viewport, 오른쪽 30%에 Properties Editor가 배치되어 있습니다.
ProteinBlender의 모든 패널은 Properties Editor의 Scene 탭(원뿔 🔻 아이콘)에 있습니다. 패널을 못 찾겠으면 페이지 하단의 패널 위치 가이드를 참고하세요.

1 단백질 구조 임포트

Protein Import 패널에서 단백질 구조를 불러옵니다. 세 가지 방법이 있습니다.

방법입력설명
로컬 파일 .pdb / .cif 폴더 아이콘 클릭 → 파일 선택. OpenFold3 결과 등 로컬 구조
RCSB PDB 4자리 PDB ID 예: 1CRN (크램빈), 6LU7 (SARS-CoV-2 프로테아제)
AlphaFold DB UniProt ID AlphaFold 예측 구조를 직접 다운로드

따라하기

  1. Properties Editor → Scene 탭(원뿔 아이콘)에서 Protein Import 패널 찾기
  2. PDB ID 입력란에 1CRN 입력
  3. "Download" 클릭
  4. 3D 뷰포트에 Cartoon 스타일로 단백질이 나타남
처음 연습할 때는 1CRN (46 잔기, 아주 작음)이 가장 좋습니다. 큰 단백질 (1000+ 잔기)은 임포트에 30~60초 걸릴 수 있습니다.
"Failed to import" 에러가 나타나면 문제 해결 가이드를 참고하세요. 가장 흔한 원인은 Blender의 "Allow Online Access" 설정이 꺼져 있는 경우입니다.

2 시각화 스타일 조정

Visual Setup 패널에서 색상과 분자 표현 방식을 바꿉니다.

스타일용도적합한 상황
Cartoon 알파 헬릭스, 베타 시트 강조 논문/발표용 전체 구조 (기본값)
Surface 분자 표면, 결합 포켓 시각화 약물-단백질 상호작용 설명
Ribbon 간략한 백본 표시 큰 복합체의 간결한 표현
Ball & Stick 원자 수준 상세 표현 활성 부위, 리간드 강조
Spheres 공간 충전 모델 분자 크기/부피 비교

따라하기

  1. Protein Outliner에서 색상을 바꿀 체인/도메인 선택
  2. Visual Setup 패널로 이동
  3. 컬러 피커에서 원하는 색상 클릭 — 별도 버튼 없이 즉시 적용됩니다 (라이브 업데이트)
  4. 표현 스타일 드롭다운에서 원하는 스타일 변경
색상이 안 바뀌는 경우? → 뷰포트 셰이딩 모드를 확인하세요!
ProteinBlender의 색상은 Geometry Nodes로 적용되므로 Solid 모드에서는 색상 변경이 보이지 않습니다.
3D Viewport 오른쪽 상단에서 세 번째 구체 아이콘 (Material Preview)을 클릭하거나, Z 키로 셰이딩 모드를 전환하세요.
도메인별로 다른 스타일을 적용할 수 있습니다. 예: 전체는 Cartoon, 활성 부위만 Ball & Stick으로 표시.

3 Puppet 만들기 (그룹핑)

Puppet은 여러 체인/도메인을 하나로 묶어서 함께 움직이게 하는 그룹입니다. 애니메이션의 핵심 단위입니다.

따라하기

  1. Protein Outliner에서 묶고 싶은 체인/도메인의 체크박스 선택
  2. Protein Puppet Maker 패널 열기
  3. "Create New Puppet" 클릭
  4. 알아보기 쉬운 이름 입력 (예: enzyme_A)
조작단축키설명
이동 (Grab)GPuppet 선택 후 마우스로 위치 이동
회전 (Rotate)RPuppet 선택 후 회전
크기 (Scale)SPuppet 선택 후 크기 조절
축 제한G→X/Y/Z특정 축으로만 이동/회전 제한
각 체인/도메인은 하나의 Puppet에만 속할 수 있습니다. Puppet에서 멤버를 빼려면 삭제 후 재생성해야 합니다.

4 Pose 저장 (컨포메이션 스냅샷)

Pose는 Puppet들의 위치/회전/크기를 저장하는 스냅샷입니다. 서로 다른 생물학적 상태를 기록해 두었다가 전환할 수 있습니다.

따라하기

  1. Puppet을 원하는 위치로 배치 (G, R, S)
  2. Pose Library 패널 열기
  3. "Create Pose" 클릭
  4. 이름 입력 (예: Open_State) → 포함할 Puppet 선택 → OK
  5. Puppet을 다른 위치로 이동 후 다시 "Create Pose" → Closed_State
버튼기능
Apply저장된 Pose로 즉시 복원
Capture현재 위치로 기존 Pose 업데이트 (덮어쓰기)
DeletePose 삭제 (Puppet은 유지됨)
생물학적 상태에 맞는 이름을 쓰세요: ATP_Bound, Substrate_Free, Transition_State 등. Pose는 Blender 파일 안에 Scene별로 저장됩니다.

5 키프레임 애니메이션

Pose 사이를 부드럽게 전환하는 애니메이션을 만듭니다. Blender가 키프레임 사이를 자동으로 보간합니다.

키프레임 삽입 방법 (2가지)

방법위치설명
방법 A (권장) Properties Editor → Scene 탭 → Animate Scene 패널 "Create Keyframe" 버튼 클릭. ProteinBlender가 선택된 Puppet의 위치/회전/크기를 현재 프레임에 자동으로 기록합니다.
방법 B (Blender 기본) 3D Viewport 위에서 I 키 팝업 메뉴에서 "Location Rotation Scale" 선택. 마우스가 반드시 3D Viewport(단백질이 보이는 영역) 안에 있어야 합니다.

기본 애니메이션 만들기

  1. 타임라인에서 프레임 1로 이동
  2. Puppet 선택 → 시작 위치(Pose) 배치
  3. Animate Scene 패널 → "Create Keyframe" 클릭 (시작점 기록)
  4. 타임라인에서 프레임 60으로 이동
  5. Puppet을 새 위치로 이동/회전
  6. 다시 "Create Keyframe" 클릭 (끝점 기록)
  7. Space로 재생하여 결과 확인
키프레임이란? "이 프레임에서 오브젝트는 이 위치/회전/크기에 있어야 한다"를 기록하는 것입니다. 두 키프레임 사이를 Blender가 자동으로 보간하여 부드러운 움직임을 만들어줍니다.

Pose 전환 애니메이션

  1. 프레임 1 → Pose Library에서 Open_State Apply → "Create Keyframe"
  2. 프레임 60 → Closed_State Apply → "Create Keyframe"
  3. 재생하면 열림→닫힘 구조 변화가 부드럽게 전환됨
키프레임 대상설명
위치 / 회전 / 크기Puppet의 3D 공간 이동
Pose 전환서로 다른 컨포메이션 간 전환
색상 변화시간에 따른 색상 전환 효과

애니메이션 재생 방법

방법설명
Space 키 마우스 커서가 반드시 3D Viewport(단백질이 보이는 영역) 위에 있어야 합니다.
사이드바나 다른 패널 위에서는 작동하지 않습니다.
타임라인 ▶ 버튼 화면 하단 타임라인 바의 재생 버튼 클릭
재생이 안 되는 경우 체크리스트:
① Shift+←로 프레임 1로 돌아가세요 (마지막 프레임에서는 움직임이 보이지 않음)
② 마우스 커서를 3D Viewport 위로 옮긴 후 Space
③ 다른 창(Preferences 등)이 열려 있으면 닫으세요
④ Preferences → Keymap → Spacebar Action이 "Play"로 설정되어 있는지 확인

반복(Cyclic) 애니메이션 설정

애니메이션이 끝나면 자동으로 프레임 1로 돌아가서 반복 재생되게 하려면 두 곳을 모두 설정해야 합니다.

① Blender 프레임 범위 설정 (필수)

Blender 자체의 재생/렌더링 범위를 키프레임 범위와 일치시켜야 합니다.

  1. 화면 오른쪽 Properties 패널에서 프린터 아이콘 (Output Properties) 클릭
  2. 맨 위 Frame Range 섹션에서 End Frame을 120으로 변경
  3. (기본값이 250이므로, 120으로 바꾸지 않으면 프레임 120 이후에도 재생이 계속됩니다)

② 반복 재생 활성화 (선택)

방법설명
ProteinBlender Animate Scene 패널 → Manual Frame Range (Start/End를 1~120) 설정 + "Cycle Animation" 체크
Graph Editor (확실한 방법) Shift+F6으로 Graph Editor 열기 →
A로 모든 커브 선택 → 메뉴 Channel → Extrapolation Mode → Make Cyclic (F-Modifier)
커브 양 끝에 점선이 표시되면 성공
ProteinBlender의 "Manual Frame Range"와 Blender의 "Frame Range"는 별개입니다!
ProteinBlender의 설정만으로는 Blender의 타임라인 End가 바뀌지 않을 수 있습니다. 반드시 ①번(Blender 프레임 범위)도 직접 설정하세요.
Output Properties의 End Frame 값이 키프레임 마지막 프레임(예: 120)과 동일한지 확인하세요.
Graph Editor (Shift+F6): 보간 곡선 정밀 제어
Dope Sheet (Shift+F2): 전체 키프레임 타임라인 관리

6 렌더링 & 출력

애니메이션이 완성되면 Blender의 렌더링 기능으로 영상을 출력합니다.

렌더링 설정

Properties Editor → Output Properties 탭 (프린터 🖨️ 아이콘)에서 아래 항목을 설정합니다.

섹션항목설정값
Format Resolution X / Y 발표용: 1920 × 1080, SNS/웹: 1280 × 720
Frame Rate 24 fps (기본값, 영화 느낌) 또는 30 fps
Frame Range Frame Start / End Start: 1, End: 마지막 키프레임 (예: 120)
Output 출력 경로 저장할 폴더 경로 지정 (폴더 아이콘 클릭)
Output
(File Format)		 File Format FFmpeg Video 선택
Color RGB (FFmpeg Video 선택 시 BW / RGB만 표시됨)
Encoding Container MPEG-4로 변경 (기본값 Matroska → MPEG-4로 바꾸세요)
Video Codec H.264 (기본값, 가장 호환성 좋음)
Output Quality Medium Quality (기본값) 또는 High Quality (논문용)
  1. 위 설정 완료 후 → 메뉴 Render → Render Animation (Ctrl+F12)
Container는 반드시 Matroska(기본값) → MPEG-4로 변경하세요!
Matroska(.mkv)는 PowerPoint/Keynote에서 재생 불가, 일부 브라우저 미지원, 학술지 투고에도 부적합합니다.
MPEG-4(.mp4)는 모든 OS, 브라우저, 발표 소프트웨어, YouTube, 학술지에서 호환됩니다.
프리뷰는 Render Engine을 EEVEE로, 최종본은 Cycles로 하면 품질과 속도를 균형 있게 조절할 수 있습니다. Mac에서는 Metal GPU 가속이 Cycles에서 잘 작동합니다.

실습 A: 첫 번째 단백질 애니메이션

예제 A-1: 크램빈(1CRN) 360도 회전

난이도: 입문 — 소요시간: ~10분 — 크램빈은 46 잔기의 작은 단백질로 연습에 최적

  1. 임포트: Protein Import 패널 → PDB ID에 1CRN 입력 → Download
  2. 색상 변경: Visual Setup에서 원하는 색상 선택 (컬러 피커 클릭 시 즉시 적용)
    ※ 뷰포트 셰이딩을 Material Preview (Z)로 전환해야 색상이 보입니다
  3. Puppet 생성: Outliner에서 체인 선택 → Create New Puppet → 이름: crambin
  4. 프레임 1: Puppet 선택 → Animate Scene → "Create Keyframe" (시작점 기록)
  5. 프레임 60으로 이동 → R → Z → 180 입력 → Enter → "Create Keyframe"
  6. 프레임 120으로 이동 → R → Z → 180 입력 → Enter → "Create Keyframe"
  7. Shift+←로 프레임 1로 돌아간 후 → Space로 재생 → 360도 회전 확인!
  8. 매끄러운 회전: Graph Editor 열기 (Shift+F6) → 회전 커브 선택 → T → Linear로 변경
왜 360도를 한 번에 입력하면 안 되나요?
Blender는 내부적으로 쿼터니언(Quaternion)을 사용하여 회전을 저장합니다. 360도 회전 = 0도 회전과 수학적으로 동일하므로, 프레임 1과 프레임 120의 회전값이 같아져서 아무 움직임도 발생하지 않습니다. 반드시 180도씩 2단계로 나누어 입력하세요.
처음에는 Ease-In/Out 보간이 기본이라 시작과 끝이 느려집니다. Linear로 바꾸면 일정한 속도로 회전합니다. 루프 영상에 적합합니다.

예제 A-2: 카메라 궤도 촬영 (Turntable)

난이도: 입문 — 단백질은 고정, 카메라가 주위를 도는 영상. 논문 Supplementary Video에 자주 활용

  1. 1CRN 임포트 (위 예제와 동일)
  2. 빈 오브젝트 추가: Shift+A → Empty → Plain Axes → 단백질 중심에 배치
  3. 카메라 부모 설정: 카메라 선택 → Shift로 Empty도 추가 선택 → Ctrl+P → Object
  4. 카메라가 단백질을 바라보도록 위치/각도 조정
    • 카메라 선택 → Numpad 0으로 카메라 뷰 확인
    • G → Z로 높이, G → Y로 거리 조절
  5. Empty 회전 애니메이션:
    • Empty 선택 → 프레임 1 → 3D Viewport 위에서 I 키 → 팝업 메뉴에서 Rotation 선택
      ※ Empty는 Puppet이 아니므로 Animate Scene 대신 Blender 기본 I 키 사용
    • 프레임 60 → R → Z → 180 → Enter → I → Rotation
    • 프레임 120 → R → Z → 180 → Enter → I → Rotation
  6. Graph Editor에서 Linear 보간으로 변경
  7. Ctrl+F12로 렌더링 → 카메라가 단백질 주위를 부드럽게 공전
카메라가 단백질을 바라보게 하려면: 카메라 선택 → Object Constraint Properties (뼈 아이콘) → Add Constraint → Track To → Target을 단백질 또는 Empty로 설정

예제 B-1: 미오글로빈 — 리간드 하이라이트

헴(Heme) 그룹이 포함된 단일 체인 단백질. 리간드만 다른 스타일로 강조하는 법을 배웁니다.

  1. 임포트: PDB ID 1MBO → Download
  2. 전체 스타일: Visual Setup → Cartoon (기본) → 색상을 연한 파란색으로 설정
  3. 헴 그룹 찾기: Protein Outliner에서 체인 구조를 확장 → 리간드/헤테로아톰(HEM) 도메인 확인
  4. 리간드 스타일 변경:
    • HEM 도메인 선택
    • Visual Setup → Ball & Stick으로 변경
    • 색상을 빨간색 계열로 변경 (컬러 피커에서 선택 시 즉시 적용)
  5. 결과: 단백질 전체는 파란 Cartoon, 헴만 빨간 Ball & Stick으로 대비
이 기법은 모든 단백질-리간드 복합체에 적용 가능합니다: 약물 분자, 보조 인자, 금속 이온 등을 다른 스타일로 구분하면 직관적인 이미지를 만들 수 있습니다.

예제 B-2: 헤모글로빈 — 멀티 체인 복합체

4개의 서브유닛(α2β2)을 각각 다른 색상으로 칠하고, 서브유닛 분리 애니메이션을 만듭니다.

  1. 임포트: PDB ID 2HHB → Download
  2. 체인별 색상 지정:
    체인서브유닛추천 색상
    Aα1파란색 (#4A90D9)
    Bβ1빨간색 (#D94A4A)
    Cα2하늘색 (#7AB8E0)
    Dβ2분홍색 (#E07A7A)
    Outliner에서 체인 하나씩 선택 → 컬러 피커에서 색상 선택 (즉시 적용)
  3. Puppet 4개 생성: 각 체인을 개별 Puppet으로 생성
    • Chain A 선택 → Create Puppet → alpha1
    • Chain B 선택 → Create Puppet → beta1
    • (C, D도 동일)
  4. "조립된 상태" Pose 저장:
    • 4개 Puppet 모두 현재 위치 그대로 유지
    • Pose Library → Create Pose → 이름: Assembled
  5. "분리된 상태" 만들기:
    • 각 Puppet을 G로 바깥쪽으로 이동 (서로 떨어지게)
    • alpha1: G → X → -3 → Enter
    • beta1: G → X → 3 → Enter
    • alpha2: G → Y → -3 → Enter
    • beta2: G → Y → 3 → Enter
    • Pose Library → Create Pose → 이름: Exploded
  6. 조립 애니메이션:
    • 프레임 1: Exploded Pose Apply → 모든 Puppet 선택 → "Create Keyframe"
    • 프레임 90: Assembled Pose Apply → "Create Keyframe"
    • 프레임 90~150: 정지 상태 유지 (같은 키프레임 복사)
  7. Space로 재생 → 4개 서브유닛이 날아와서 조립되는 애니메이션!

예제 B-3: SARS-CoV-2 프로테아제 — 활성 부위 줌인

효소의 전체 구조를 보여준 후 활성 부위로 카메라가 줌인하는 애니메이션.

  1. 임포트: PDB ID 6LU7 → Download
  2. 시각화:
    • 전체: Cartoon, 연한 회색 (#B0B0B0)
    • 리간드(N3 억제제): Ball & Stick, 노란색 (#FFD700)
    • 활성 부위 잔기(His41, Cys145 근처 도메인): Cartoon, 초록색 (#50C878)
  3. 카메라 설정:
    • 카메라를 전체 구조가 보이는 원거리 위치에 배치
    • 프레임 1: 카메라 위치 키프레임 (3D Viewport 위에서 I → Location)
      ※ 카메라는 Puppet이 아니므로 Blender 기본 I 키 사용
  4. 줌인 카메라 이동:
    • 프레임 60: 카메라를 활성 부위 근처로 이동 (G로 전진)
    • 카메라 뷰(Numpad 0)에서 리간드가 화면 중앙에 크게 보이도록 조정
    • 3D Viewport 위에서 I → Location 키프레임 삽입
  5. 프레임 60~120: 줌인 상태에서 카메라를 약간 회전하여 활성 부위를 다양한 각도로 보여줌
  6. Graph Editor에서 줌인 구간의 보간 곡선을 Ease-In으로 설정 → 자연스러운 감속
카메라 Depth of Field(피사계 심도) 추가: Camera Properties → Depth of Field → Focus Object를 리간드로 설정 → F-Stop 0.5~2.0. 배경이 흐려지면서 활성 부위에 시선이 집중됩니다.

예제 B-4: 스파이크 단백질 — 도메인 분리 (Exploded View)

SARS-CoV-2 스파이크의 3량체(trimer) 구조를 도메인별로 분리하는 교육용 애니메이션.

  1. 임포트: PDB ID 6VYB → Download (약 30~60초 소요, 큰 구조)
  2. 3개 체인 색상 구분:
    • Chain A: 초록색, Chain B: 주황색, Chain C: 보라색
    • 각 체인을 Outliner에서 선택 → 색상 적용
  3. Puppet 생성: 각 체인(모노머)을 개별 Puppet으로 생성
  4. Pose 2개 저장:
    • Trimer_Intact: 원래 위치 그대로
    • 각 Puppet을 중심에서 바깥으로 G → 방사형으로 이동
    • Trimer_Separated: 분리된 위치 저장
  5. 타임라인 구성:
    프레임상태설명
    1~30Trimer_Intact전체 구조를 보여줌
    30~90전환3개 모노머가 서서히 분리됨
    90~150Trimer_Separated분리 상태에서 회전하며 각 모노머 확인
    150~210전환다시 합쳐짐
    210~240Trimer_Intact마무리
  6. 각 전환 구간의 시작/끝에 키프레임 삽입
스파이크 단백질은 약 3,000+ 잔기로 무겁습니다. EEVEE 엔진으로 프리뷰하고, 뷰포트 셰이딩을 Solid로 유지하면 작업이 훨씬 빠릅니다. 최종 렌더링만 Cycles + Material Preview로 전환하세요.

예제 B-5: OpenFold3 예측 구조 — pLDDT 신뢰도 시각화

OpenFold3-MLX로 예측한 구조를 임포트하여 신뢰도에 따른 색상을 부여하고 논문 이미지를 생성합니다.

  1. 구조 예측: OpenFold3-MLX GUI 또는 CLI로 시퀀스 예측 실행
    • 결과 파일: output/prediction.pdb (B-factor 컬럼에 pLDDT 점수 저장됨)
  2. 임포트: Protein Import 패널 → 폴더 아이콘 → prediction.pdb 선택
  3. pLDDT 색상 스킴 적용: 도메인/영역별로 수동 색상 지정
    pLDDT 범위의미색상Hex
    > 90매우 높은 신뢰도진한 파란색#0053D6
    70 ~ 90높은 신뢰도하늘색#65CBF3
    50 ~ 70낮은 신뢰도노란색#FFDB13
    < 50매우 낮은 신뢰도주황색#FF7D45
    (이 색상은 AlphaFold DB 공식 컬러 스킴과 동일)
  4. 비교 이미지 제작:
    • 같은 구조를 두 번 임포트 → 하나는 Cartoon, 하나는 Surface
    • 나란히 배치하여 2D 구조 vs 3D 표면을 동시에 보여줌
  5. 논문용 정적 이미지 렌더링:
    • 카메라 앵글 잡기 → Numpad 0으로 확인
    • Render → Render Image (F12) → Image → Save As로 PNG 저장
    • 해상도: 최소 300 DPI (3000x2000 px 이상 권장)

부록: 문제 해결 (Troubleshooting)

"Failed to import" 에러

Download 버튼을 눌렀을 때 Blender 하단 상태 바에 Failed to import: 1CRN 같은 에러가 표시되는 경우:

1. Allow Online Access 확인 (가장 흔한 원인)

Blender 4.2+에서는 확장 프로그램의 인터넷 접근이 기본적으로 제한됩니다.

  1. Edit → Preferences (macOS: Blender → Settings)
  2. 왼쪽 메뉴에서 System 클릭
  3. Network 섹션에서 "Allow Online Access" 체크박스 활성화
  4. Preferences 창 닫기 → Download 재시도

2. 실제 에러 메시지 확인 (macOS)

ProteinBlender는 에러 상세 메시지를 콘솔에 출력하지만, macOS에서는 기본적으로 보이지 않습니다.

  1. Terminal 앱 열기 (Spotlight에서 "Terminal" 검색)
  2. 아래 명령어로 Blender 실행:
    /Applications/Blender.app/Contents/MacOS/Blender
  3. ProteinBlender에서 Download 다시 시도
  4. Terminal에 출력된 에러 메시지 확인 — Failed to import PDB 1CRN: ... 형태

3. 포맷 변경 시도

기본 포맷(PDB)으로 다운로드가 안 되면, Protein Import 패널에서 포맷을 mmCIF로 변경 후 재시도해 보세요. RCSB에서 일부 구조는 PDB 포맷이 제공되지 않을 수 있습니다.

4. MolecularNodes 업데이트 (databpy 충돌)

Blender 확장 목록에 MolecularNodes가 별도로 설치되어 있으면 databpy 버전 충돌이 발생할 수 있습니다. Terminal에 ImportError: cannot import name 'Domains' from 'databpy' 에러가 보이면:

  1. Edit → Preferences → Get Extensions
  2. MolecularNodes를 최신 버전으로 업데이트
  3. Blender 재시작 후 Download 재시도

ProteinBlender는 MolecularNodes를 내장하고 있어 별도 설치가 필수는 아니지만, 이미 설치되어 있다면 최신 버전으로 유지해야 충돌이 없습니다.

5. ProteinBlender 재설치

위 방법으로도 해결되지 않으면 ProteinBlender 확장을 제거 후 재설치해 보세요: Edit → Preferences → Get Extensions → Installed → ProteinBlender 제거 → 재설치
